arxiv.submission.tests.classic.test_classic_integration module¶

Tests for integration with the classic system.

Provides test cases for the new events model’s ability to replicate the classic model. The function TestClassicUIWorkflow.test_classic_workflow() provides keyword arguments to pass different types of data through the workflow.

class arxiv.submission.tests.classic.test_classic_integration.TestWithdrawalIntegration(methodName='runTest')[source]¶

Bases: unittest.case.TestCase

Test integration with the classic database concerning withdrawals.

The domain.submission.Submission representation has only two statuses: domain.submission.WITHDRAWAL_REQUESTED and domain.submission.WITHDRAWN. Like other post-publish operations, we are simply adding events to the single stream for the original submission ID. This screens off details that are due to the underlying implementation, and focuses on how humans are actually interacting with withdrawals.

On the classic side, we create a new row in the submission table for a withdrawal request, and it passes through the same states as a regular submission.
