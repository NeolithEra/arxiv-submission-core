agent.runner.async module¶

Provides an asynchronous process runner.

The AsyncProcessRunner registers and dispatches processes via Celery, to be carried out by the worker.

class agent.runner.async.AsyncProcessRunner(process)¶

Bases: agent.runner.base.ProcessRunner

Runs Process instances asynchronously.

In order for this to work at runtime, prepare() MUST be called with the process class at import time in both the process that dispatches tasks (e.g. the event consumer) and the worker process.

agent.runner.async.make_countdown(delay, backoff=1, max_delay=None, jitter=0)¶

Make a countdown callable based on the retry parameters of a step.

For use in task retry calls, to customize the retry delay.

Parameters

  • delay (int) – The base number of seconds to wait before retrying.

  • backoff (int) – If provided, the exponent applied to the delay * number of attempts.

  • max_delay (int) – If provided, the maximum number of seconds to wait.

  • jitter (int or tuple) – If an int, number of seconds (+/-) to alter the delay, after the backoff is applied. If a two-tuple of ints, a random offset will be used in the range (jitter[0], jitter[1]).

Return type

Callable[[int], Union[int, float]]

Returns

function – Countdown function; accepts a single int parameter representing the current retry count.
