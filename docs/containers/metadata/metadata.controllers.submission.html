metadata.controllers.submission.handlers module¶

Data handling logic for submission data payload.

Each handler (a function) should accept two parameters:

  • data is the value of the field that it handles. This can be anything that is deserializable from a JSON document.
  • agents is a dict with the creator, proxy, and client agents to use when creating new events.

The primary controller in this module is the handle_submission(), which delegates work to the handlers. The global HANDLERS defined at the end of this module describes how delegation should occur.

Note: data validation should not be implemented here! Events/commands in events should define required parameters, perform all validation, and carry out any required transformation/cleanup.
